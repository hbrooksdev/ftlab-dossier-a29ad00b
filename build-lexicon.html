Canonical Lexicon (frozen)

The single authoritative definition of FTLAB’s load-bearing terms — the spec every charter and script compiles against. Six contradictions resolved; org units = Core/Engines/Edge.

5 min readClassification · Strategic / InternalSource · 00-canonical-lexicon-v1.0.md
In brief

Frozen first because a Core that can be built will compile a wrong spec as readily as a right one (“buildable ≠ correctly specified”).

A. The architecture (canonical organisational units)

FTLAB is a Core, ~3 Engines, and an Edge — the determined architecture (Option 3 path → Option 2 / Mothership destination). The earlier "seven Domains" is retired as the organisational unit; it survives only as historical detail.

TermCanonical definition
The Machine Core ("the Core")The engineered coordination substrate every Engine runs on: the institutional memory + the integrator (routing), the Strategy-as-Code repository, the Agent Charter registry, the IPRE pipeline, the Governance Loop (Alignment Debt + Risk Twins), decision-tracing, and the evaluator-agent layer. It carries the routing job that hierarchy used to. It is also, turned outward, the licensed Platform product.
An EngineA value-creating function presented as a named, accountable VP-Agent ("Head") orchestrating an agent team, coordinating through the Core (not a manager), and escalating to the Edge. There are three: see below.
The EdgeThe only humans: the founders, holding the New Triumvirate (Architect / Intent-Setter / Guardian) over the agents. Holds intent, capital allocation, values authorship, and the final halt. The job AI does not dissolve.
The three Engines(1) The Venture Engine — research-through-venture-building + incumbent transformations as co-ventures. (2) Platform & Agentic Systems — the AI-Native Enterprise Framework FTLAB licenses, and the agent infrastructure (≈ the Core, turned outward). (3) Knowledge & Ecosystem — publications, education, and the GCC/sovereign/partner relationships.
Coordination & MemoryThe Core's institutional memory + the integrator (the routine-coordination agent that routes work and sequences hand-offs). Not an Engine — it is the Core. (In the retired Option-1 framing it was a department, "D2".)
Capital & OperationsHeld by the Edge (the founders) until it warrants its own seat. Not an Engine.
The Knowledge FlywheelResearch → thesis → venture → data → research. Operationalised as cross-venture learning (shared agent library, federated learning) running on the Core.

The number is a floor, held as an evolving shape, not a template. It widens only as the work demands (founders' "principles and logics, not fixed structures").

B. The six frozen resolutions

1 · The human-to-agent ratio — no fixed ratio is load-bearing. The principle: a small human Edge (single digits to low tens) governs a large agent population, each Engine's population sized by risk, reversibility, and regulatory intensity — tighter where regulated/trust-bearing, larger and more autonomous where low-stakes and reversible. "5:100" is an external, illustrative shorthand only, flagged as non-binding.

2 · "Five Planes" = the technical stack only — Data, Model, Agent, Orchestration, Actuation. The Five Planes live inside the Core. The organisational units are the Core / Engines / Edge (above). Any use of "Five Planes" to mean organisational layers is retired.

3 · "The New Triumvirate" = Architect / Intent-Setter / Guardian — three governance functions held by the Edge, in deliberate tension, never concentrated. (The "Human Judgment / Machine Intelligence / Institutional Design" triad is a different concept — sources of capability — and must be named differently if used.)

4 · "VP-Agent" = a Virtual Professional bounded by Values & Preferences. Virtual Professional names the role (the Head agent that owns an Engine or a Core function); Values & Preferences names how its authority is bounded — Values are non-negotiable constraints (the hardConstraints), Preferences are tunable, with structural preferences in between. Always introduce the pair together.

5 · Phantom agents — removed. "Platform Architect" → the Platform & Agentic Systems Engine. "Deal Scout" → the Venture Engine (scouting/origination), with origination itself sitting at the Edge (the Intent-Setter), per the determination's gating item #5. data-engineer and qa-engineer re-point to their Engine (Platform; Quality/oversight in the Core).

6 · Leadership titles — Managing Partner (Shayan) / General Partner (Mehran) internally. "CEO" connotes a command hierarchy the model rejects; CEO-equivalents are retained only where external/DIFC legibility requires them.

C. Operating terms (canonical)

  • Agent Charter — the machine-readable constitution of a single agent (see 00-agent-charter-schema-v1.0.md); its hardConstraints block is architecturally prior to its reward function. A charter precedes its agent; never the reverse.
  • Strategy-as-Code — organisational strategy encoded precisely enough to compile, version-controlled, testable; lives in the Core's repository. "If governance isn't in the repository, it is aspiration."
  • IPRE pipeline — Intent → Plan → Run → Evaluate. The Plan stage is the last human checkpoint before autonomous execution.
  • PRAL loop — Perceive → Reason → Act → Learn (per-agent execution cycle).
  • Governance Loop — Edge encodes intent as Strategy-as-Code → IPRE executes → Alignment Debt monitoring surfaces drift → Risk Twins validate changes before deployment → Edge updates the repo.
  • Alignment Debt — five drift metrics: policy-violation-rate trend, intent-outcome gap, evaluator disagreement, escalation suppression, reward hacking.
  • Risk Twins — simulation of a proposed strategy change against historical + adversarial scenarios before production; a deployment gate.
  • Override tiers — the three decision-rights tiers in every charter: autonomous / confirmation-required / human-escalation.
  • The Guardian halt-right — the Guardian's power to stop, redirect, or constrain any agent on values grounds; carries no P&L (see 00-guardian-and-independence-v1.0.md).
  • Iteration Half-Life (IHL) — time from strategic intent to measurable impact; FTLAB's primary operating metric.
  • Cognitive Overhead Index (COI) — diagnostic of how much human attention the system consumes; < 30 ≈ genuinely AI-born.

This lexicon is the Core's contract. Phase 0 also delivers the charter schema, the Values-as-code, and the Guardian/independence charter; all three compile against the terms above. The master build plan is ROLLOUT-v1.0.md.
