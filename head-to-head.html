The determination · verdict

The Head-to-Head (v0.3)

The integrated verdict after Book 1 grounding and the retirement of the build-time objection — the recommendation reversed from v0.2.

5 min readClassification · Strategic / InternalSource · ftlab-org-architecture-head-to-head-v0.3.md
In brief

The v0.3 comparison. Re-grounded in Book 1 and with build-time no longer an objection, the earlier Option-1-led hedge inverts: build the Core (the Option 2 destination) by the Integrated path (Option 3); Option 1 is not a destination; Option 4’s market is rejected.

Confirmed by hostile pressure-tests of all four, and corrected for two over-claims (soundness vs shape; two internal markets, not one).

Key takeaways
  • Shape confirmed under attack: Option 3 path → Option 2 destination; not Option 1; not Option 4’s market.
  • Removing build-time settles the shape, not the soundness — and sharpens “buildable ≠ correctly specified.”
  • The gating sequence: Guardian independence → freeze + red-team the spec → external accountability terminus → fund the thinning → move origination out of the Core.
  • The structure, derived from the work, is ~3 Engines + Core + Edge.

What changed since v0.2, and why

v0.2 recommended an Option 1-led posture: ship the familiar seven-domain face now, build the Core on a separate, red-teamed, timeboxed track, and treat Option 2/3 as the eventual destination and Option 4 as a far-future hypothesis. That recommendation rested on one load-bearing assumption: that the Core is a long, risky, pre-value build, so deploying the familiar shape first was the prudent hedge.

Two inputs changed that.

  1. The architecture was re-grounded in Book 1 (AI-Born: The Machine Core), using the chapters that actually bear on org design — Ch 3, 4, 5, 6, 8, 9, 10 (Ch 7 partial; Ch 1–2 directional only). Book 1 and FTLAB's framework turn out to be one intellectual system, and Book 1 hands over a concrete build-spec for the Core (Ch 5–6) and shows that FTLAB itself is a Mothership (Ch 10), which is Option 2 component-for-component. Full reasoning: ftlab-architecture-determination-book1-grounded-v0.1.md.
  2. The build-time objection was retired by instruction: as long as the Core can be built, the design is valid. Buildability and build-duration are no longer grounds to reject a design.

Remove the build-time hedge and Option 1's only advantage disappears. The verdict reverses.

The four options (one line each)

  • Option 1 — Seven Domains. A department model: seven functional domains, each an AI Head with an agent team, governed by the human Cortex. Familiar, legible.
  • Option 2 — Core · Engines · Edge. A shared coordination substrate (the Core), functional Engines (VP-Agents), and a thin Edge (the human Cortex + venture surface). The Mothership.
  • Option 3 — The Integrated. Option 1's familiar functional face worn over Option 2's real Core, from day one, with the face thinning as the Core matures.
  • Option 4 — The New Mode. A constitutional core plus an internal market/ecology of venture-cells selected by a fitness signal, sharing a genome.

The verdict

Build the real Machine Core to the Book 1 (Ch 5–6) specification, present it through a small set of familiar domain-faces (the Engines), and govern it through a thin Human-Cortex Edge running the New Triumvirate. This is Option 3 as the path, converging on Option 2 / the Mothership as the destination. Not Option 1 as an endpoint; not Option 4's internal market.

This was confirmed under hostile pressure-testing of all four options (*-pressure-test-*-v0.2.md), which attacked the verdict directly rather than defending it.

What the pressure-tests corrected in the determination

The tests confirmed the shape and caught two over-claims, now fixed in the determination (...determination...-v0.1.md §9):

  • Removing build-time settles the shape, not the soundness — and it sharpens the #1 real risk. A Core that can be built compiles whatever spec it is given; FTLAB's own canonical-terms work shows the load-bearing primitives are still defined inconsistently across the corpus. The danger moves from "we never finish the Core" to "we finish a confidently wrong Core," and the verifier can't catch it because it checks against the spec.
  • There are two internal markets, not one. The Ch 10 + statistical-power argument defeats the venture-cell selection market; a distinct agent-task market can clear but loses on collusion-economics and redundancy with the Core's routing. Both rejected, different defeaters.

The gating sequence — the work that follows the decision

Choosing the shape does not make it safe. Every shape inherits #1; #2–#5 are conditions on building the determined shape well. In priority order:

  1. Independent Guardianship at 2–5 humans (founding-team capture). A Guardian with no P&L plus an external Guardian-auditor with a real, exercised halt-right. Independence cannot come only from inside the Cortex. Solve first.
  2. Freeze and red-team the specification for correctness before the Core acts autonomously on anything consequential.
  3. An external accountability terminus with teeth — board, auditor, or sovereign counterpart able to compel a halt. Counters the Oligarch's Temptation that the Core concentrates and Strategy-as-Code makes durable.
  4. Fund the thinning of the faces — symmetric with widening: every gate that widens Core autonomy withdraws the matching Engine mandate; failure-to-thin logs as Alignment Debt that brakes further widening; name an owner of "face operational weight."
  5. Move origination out of the Core — keep it with the human Intent-Setter, so the Cortex directs rather than ratifies.

Bottom line

The Book 1 grounding and the retirement of the build-time objection turn the earlier hedge inside out. There is no longer a reason to ship the AI-enabled shape (Option 1) as a destination, and there is a fully specified Core to build instead. FTLAB should build the Mothership it already is — a real Core, familiar faces over it, a thin Triumvirate Edge — on the Option 3 path toward the Option 2 destination. The decision is the easy part; the five-item gating sequence is the hard part, and the first item — genuine, externally-anchored Guardianship for a 2–5-person founding Cortex — must be solved before the Core is trusted with anything that matters.
