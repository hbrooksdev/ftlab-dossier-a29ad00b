The Machine Core — Strategy-as-Code Repository

Status: Internal / Strategic / Confidential · 2026-06-01 · Phase 1 Compiles against: ../00-canonical-lexicon-v1.0.md, ../00-agent-charter-schema-v1.0.md, ../00-values-as-code-v1.0.md

This directory is the Core — the engineered coordination substrate every Engine runs on. It is governed as code: precise, version-controlled, testable. If governance isn't in the repository, it is aspiration (Book 1 Ch 6).

Structure

core/
├── README.md                         this file
├── coordination-and-memory.md        the integrator: memory schema + routing rules + its charter
├── ipre-pral-and-governance-loop.md  the operating spine (IPRE, PRAL, override tiers, tracing, Governance Loop)
├── charters/                         the charter registry (one YAML per agent; Phase 2 fills it)
│   └── REGISTRY.md                   index of registered charters + their risk class + Guardian sign-off
├── decisions/                        the immutable decision trace
│   ├── decision-log.schema.json      the data contract every consequential decision writes
│   └── sample-decisions.json         a fixture so the oversight scripts run today
├── policies/                         Strategy-as-Code (the Intent-Setter's tunable strategy; Tier-3 preferences)
└── scripts/                          the oversight harness (override-rate, random-audit, alignment-debt)

How the Core works (one paragraph)

The Edge encodes intent as Strategy-as-Code (policies/). The IPRE pipeline turns it into chartered work; agents execute their PRAL loop within their charter's three decision tiers. Every consequential action writes a record to the decision trace (decisions/). The Governance Loop reads that trace: the oversight scripts compute Alignment Debt and the Guardian's dashboards; Risk Twins validate strategy changes before deployment. The integrator (coordination-and-memory.md) routes work between agents through shared memory rather than a management chain — and escalates anything contested to the Edge.

Build order (Phase 1)

  1. decisions/decision-log.schema.json — the data contract (built first; everything else writes to it).
  2. coordination-and-memory.md — the integrator + memory schema + routing rules.
  3. ipre-pral-and-governance-loop.md — the spine.
  4. scripts/ — the oversight harness, runnable on sample-decisions.json.
  5. charters/REGISTRY.md — the registry (Phase 2 populates it with Engine charters).

The honest limit at this stage (Option 3)

Until the Core's routing has earned reliability, the integrator is human-supervised on contested routing: it routes and surfaces, but contested cross-Engine or values-level questions escalate to the Edge (it never adjudicates them). Autonomy widens by measured reliability, never by date. This is the "face, not yet thinned" stage — exactly as the determination describes Option 3 early on.
