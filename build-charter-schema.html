Agent Charter Schema

The 7-element machine-readable charter every agent compiles to — hardConstraints prior to reward, three decision tiers, the YAML template, a worked sketch, and the five compile rules.

4 min readClassification · Strategic / InternalSource · 00-agent-charter-schema-v1.0.md
In brief

A charter precedes its agent; never the reverse.

The seven elements

  1. Identity & classification — name; whether it is an Engine Head, a team agent, or a Core agent; its Engine/Core home; model tier; and a risk class (R1 reversible/low-stakes … R4 irreversible/regulated/trust-bearing) that sets default oversight.
  2. Role definition — the mandate in one sentence; what it owns; what it explicitly does not own (single-owner rule).
  3. Decision rights (three tiers) — autonomous (acts and logs), confirmation-required (a peer agent or Head must confirm), human-escalation (the Edge decides). Each tier is defined by explicit thresholds.
  4. Forbidden actions — the "never" line. Bright-line actions the agent may not take under any reward (these reference the Values-as-code hardConstraints).
  5. Objective function — the reward weights and the constraint metrics. Stated so that "maximise X" can never run unbounded — every objective carries its bounding constraints inline.
  6. Escalation policy — the triggers that send a decision to a human regardless of confidence: irreversibility, novelty (outside trained distribution), material stakeholder impact, an unresolved values conflict, regulatory ambiguity — plus numeric thresholds.
  7. Performance monitoring — the metrics the Core tracks for this agent, its evaluator-agent hooks, and what it contributes to Alignment Debt (override rate, intent-outcome gap, escalation-suppression signal).

The template (YAML)

yaml
charter_version: "1.0"
compiles_against: "00-canonical-lexicon-v1.0.md"

identity:
  name: ""                 # e.g. venture-engine-head
  kind: ""                 # engine_head | team_agent | core_agent
  home: ""                 # venture-engine | platform-agentic-systems | knowledge-ecosystem | core | edge
  model_tier: ""           # the model class this agent runs on
  risk_class: ""           # R1 | R2 | R3 | R4

role:
  mandate: ""              # one sentence: what this agent is for
  owns: []                 # the functions it is the single accountable owner of
  does_not_own: []         # explicit boundaries (prevents matrixed ownership)

# hardConstraints are checked BEFORE the reward function. Inviolable.
hard_constraints:
  refs: []                 # ids from 00-values-as-code-v1.0.md (e.g. V1, V3, V7)
  inline: []               # any agent-specific bright lines

decision_rights:
  autonomous:              # acts and logs; no confirmation
    conditions: []         # e.g. ["risk_class in [R1,R2]", "spend < 500"]
  confirmation_required:   # a peer agent or the Head must confirm
    conditions: []
  human_escalation:        # the Edge decides
    conditions: []

forbidden_actions: []      # the "never" line — bright-line prohibitions

objective_function:
  maximise: []             # reward terms
  subject_to: []           # constraint metrics bounding each reward term
  notes: ""

escalation_policy:
  triggers: [irreversibility, novelty, material_stakeholder_impact, values_conflict, regulatory_ambiguity]
  thresholds: {}           # e.g. {confidence_below: 0.75, deal_value_over: 50000, regulatory_flag: true}
  escalates_to: ""         # the Triumvirate function (architect | intent_setter | guardian)

monitoring:
  metrics: []              # what the Core tracks
  evaluator_hooks: []      # which evaluator agents watch this one
  alignment_debt_signals: [override_rate, intent_outcome_gap, escalation_suppression]

provenance:
  author: ""               # who wrote the charter (the Edge function)
  approved_by: ""          # the Guardian sign-off for R3/R4 agents
  registered_in_core: false

Worked sketch — the Venture Engine Head (illustrative)

yaml
identity: { name: venture-engine-head, kind: engine_head, home: venture-engine, risk_class: R3 }
role:
  mandate: "Turn theses into ventures and operate them through the pipeline; run incumbent transformations as co-ventures."
  owns: ["thesis origination support", "venture design", "venture operation (Thesis→Prototype→Named→Growth)"]
  does_not_own: ["capital commitment", "final partner terms", "values adjudication"]
hard_constraints: { refs: [V1, V2, V5, V7] }            # honesty-to-partners, no regulated activity, etc.
decision_rights:
  autonomous: { conditions: ["stage in [Thesis,Prototype]", "no external commitment"] }
  confirmation_required: { conditions: ["stage-gate promotion", "new agent role"] }
  human_escalation: { conditions: ["capital commitment", "partner/venture terms", "irreversible action"] }
forbidden_actions: ["commit FTLAB capital", "sign a partner without the Edge", "mislead or conceal from a partner"]
objective_function:
  maximise: ["validated theses → ventures reaching product-market fit"]
  subject_to: ["Iteration Half-Life < target", "no hard_constraint violations", "honest exit criteria respected"]
escalation_policy: { escalates_to: intent_setter }

The Venture Engine carries life-stage sub-charters (Thesis / Prototype / Named / Growth), each a narrower envelope of this one, so the single seat still compiles rather than becoming an unbounded grant.

Compile rules (enforced before an agent goes live)

  1. One accountable owner. role.owns is non-empty and disjoint from every other charter's owns.
  2. hard_constraints resolve. Every refs id exists in 00-values-as-code-v1.0.md.
  3. Tiers are total. The three decision-rights tiers cover every action class the mandate implies; nothing falls through.
  4. R3/R4 require Guardian approval (provenance.approved_by) before registered_in_core: true.
  5. Registered in the Core. The charter lives in the Strategy-as-Code repository and is version-controlled; changes are traced.

Next foundation files: 00-values-as-code-v1.0.md (the hard_constraints these charters reference) and 00-guardian-and-independence-v1.0.md (who approves R3/R4 and holds the halt).
