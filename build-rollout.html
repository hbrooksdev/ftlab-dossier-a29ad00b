The build · runbook

Master Rollout (v1.0)

The single plan the org is built against — five phases for the Core + ~3 Engines + Edge, the agent reconciliation map, the gating preconditions, and rollback. Retires the two old rollouts.

5 min readClassification · Strategic / InternalSource · ROLLOUT-v1.0.md
In brief

Implementation of the determination, in the order the gating sequence sets: freeze the foundation → build the Core → stand up the three Engines → document → go live on one venture.

The five phases

Phase 0 — Freeze the foundation (gating · this directory)

Done: the canonical lexicon (v1.0, frozen), the Agent Charter schema, the Values-as-code (the genome), the Guardian charter + independence plan. Exit criteria:

  • Lexicon applied to CLAUDE.md (Five Planes → stack-only; org units → Core/Engines/Edge; Triumvirate; VP-Agent; titles).
  • Guardianship Phase-0 exit criteria met (see 00-guardian-and-independence-v1.0.md §6).
  • Values-as-code reviewed by the Edge (it is the genome — a values-authorship act).

Phase 1 — Build the Core scaffolding

  1. Strategy-as-Code repository — a core/ tree: lexicon/, values/, policies/, charters/ (the registry), decisions/ (the trace). Version-controlled; changes traced.
  2. Coordination & Memory (the integrator) — reconcile rollout/agents/knowledge-memory-coordination.md + the knowledge-manager, framework-architect, research-architect, agent-orchestrator agents into one Core memory + routing service. Built first, devolved last. Honest limit (Option 3): initially human-supervised on contested routing.
  3. IPRE pipeline + PRAL loop + override tiers + decision-tracing — the Intent→Plan→Run→Evaluate spine (Plan = last human checkpoint) and the per-agent loop; the three decision-rights tiers; the immutable audit trail.
  4. The oversight harness (scripts) — the evaluator-agent pattern, the override-rate dashboard, the weekly random-audit sampler, and the Alignment-Debt metric collectors. (Risk Twins follow once there is history to simulate against.)

Verify: the integrator can route a real task across two agents and surface a conflict to the Edge; the audit trail reconstructs a decision end-to-end; the random-audit sampler runs. Rollback: the Core is additive; if the integrator misroutes, fall back to direct Edge assignment (it is human-supervised by design at this stage).

Phase 2 — Stand up the three Engines

For each Engine: write the Head charter (VP-Agent), reconcile the relevant existing agents into its team (each with a charter), wire it to the Core (coordination) and the Edge (escalation). Reconciliation map in §B below.

  • Venture Engine — Head charter + life-stage sub-charters (Thesis/Prototype/Named/Growth) + team.
  • Platform & Agentic Systems — Head charter + team.
  • Knowledge & Ecosystem — Head charter + team.

Verify: every charter compiles (the five rules in the schema); R3/R4 agents carry Guardian approval; no two charters own the same function. Rollback: Engines deploy one at a time; each is reversible by de-registering its charters from the Core. Keep the existing 28 agents available until the Engines verify.

Phase 3 — Document + operating manual + venture template

  • The org operating manual (how the Core, Engines, and Edge run day-to-day; the IPRE cadence; the Guardian cadence).
  • Update the venture-org template to the fractal ~3-Engines + Core + Edge (each venture instantiates the same shape at its own scale).
  • Apply the lexicon to CLAUDE.md and reconcile/retire the phantom agents.

Phase 4 — Go live on one venture + instrument

  • Stand up on one venture first (Adaptic is the live one per the founder responses), not the whole portfolio.
  • Instrument Iteration Half-Life, Cognitive Overhead Index, and Alignment Debt from day one.
  • Pre-write the honest exit criteria (Book 1 Ch 10): if three consecutive ventures miss profitability in 24 months, review the model, not just the execution.
  • Widen agent autonomy by measured reliability, never by date.

A. Gating preconditions (must hold before the Core acts autonomously on R3/R4)

  1. Lexicon frozen + applied (Phase 0). ✔ frozen; applying to CLAUDE.md is a Phase-3 step but the freeze governs from now.
  2. Guardian independence — external halt-right mechanism specified; R3/R4 pinned at human-escalation until it is live (00-guardian-and-independence-v1.0.md).
  3. Values-as-code authored by the Edge — the genome is a values-authorship act, not an agent output.
  4. Origination stays at the Edge — the Venture Engine supports origination; the Intent-Setter owns the decision (gating #5).

B. Agent reconciliation map (first cut — Phase 2 finalises)

The existing 28 agents + the 7 staged heads reconcile as follows. (Heads in bold; this is the first cut, refined when each Engine's charters are written.)

Venture Engineresearch-director (Head candidate) + synthesis-agent, research-scout, strategy-analyst, research-portfolio-manager, research-architect, venture-architect; absorbs rollout/agents/{research-intelligence, venture-building}
Platform & Agentic Systemsdata-engineer / platform lead (Head) + engineering + security sub-agents (new); absorbs rollout/agents/platform-agentic-systems and the phantom "Platform Architect"
Knowledge & Ecosystemeditorial-director (Head candidate) + creative-director, education-designer, content-distribution-manager, technical-writer (comms), investor-relations, founder-partner, partnership-development, community-manager, event-coordinator; absorbs rollout/agents/communication-ecosystem

Core — oversight machinerycritical-reviewer, qa-engineer (evaluator/validation); absorbs rollout/agents/quality-guardianship (machinery only)
Edge — governance (human + adjudication)the founders (Architect/Intent-Setter/Guardian) + compliance-officer, legal-counsel as advisory agents the Guardian governs
Edge — Capital & Operationschief-of-staff, people-operations, vibe-director; absorbs rollout/agents/capital-people-operations — Edge-held until it earns a seat

The Option-2 staged heads (rollout-option2/agents/) inform the Core's internal structure (router, verification-layer) and the Knowledge Flywheel naming; they are not deployed as-is.

C. Rollback discipline (whole rollout)

  • The Core is built before the org depends on it; the boundary layer (what the model informs vs. what a human decides) is written before the model is trusted.
  • Every phase is reversible: Core additive; Engines de-registrable; the 28 agents stay until their replacements verify.
  • Widen autonomy by evidence (reliability, COI, override-rate), never by calendar.
  • The Guardian halt-right and the V-tier hardConstraints are live from the first agent action.

This runbook is the single source for the build. Phase 0 is complete in this directory; Phases 1–4 add their artifacts here (core/, engines/, docs/). Strategy: ftlab-architecture-determination-book1-grounded-v0.1.md; verdict: ftlab-org-architecture-head-to-head-v0.3.md.
